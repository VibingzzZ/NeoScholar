**第1页：核心概念 - Agent的感知与行动**
- **标题：** Agent的“大脑”与“肢体”
- **要点：**
    - **感知：** Agent如何“看”和“听”世界。
        - 核心：将外部信息（文本、图像、数据）转化为内部可处理的表示。
    - **行动：** Agent如何“动手”和“动脚”。
        - 核心：执行特定任务，改变外部环境或获取新信息。
    - **本节要点：** 聚焦于两大核心能力：
        - **感知增强：** 通过RAG实现知识检索。
        - **行动扩展：** 通过工具调用（Web浏览、文件操作）与环境交互。

**第2页：感知升级 - 集成RAG实现知识检索**
- **标题：** 让Agent拥有“长期记忆” - RAG详解
- **要点：**
    - **RAG（检索增强生成）**：解决大模型知识过时、幻觉问题。
    - **工作流程：**
        1.  **检索：** 根据用户问题，从外部知识库中检索相关文档片段。
        2.  **增强：** 将检索到的文档片段作为上下文，与用户问题一起提供给大模型。
        3.  **生成：** 大模型基于提供的上下文生成更准确、可信的回答。
    - **示例：** 用户问“公司去年的营收是多少？” -> Agent从内部文档库检索到相关财报 -> 生成精确答案。

---

**第3页：感知基石 - 向量数据库与嵌入模型**
- **标题：** 知识检索的“搜索引擎” - 向量数据库与嵌入
- **要点：**
    - **嵌入模型：** 将文本（如文档、查询）转换为数值向量。
        - 作用：捕捉语义相似性。
        - 示例：`text-embedding-ada-002`（OpenAI）、`BAAI/bge-small-en-v1.5`（开源）。
    - **向量数据库：** 存储和检索这些高维向量的专用数据库。
        - 作用：高效地进行语义相似度搜索（例如，找到最相关的10个文档片段）。
        - **推荐工具：** **ChromaDB**（轻量级、易上手、适合原型开发）。
    - **集成方式：** 用户提问 -> 嵌入模型转换 -> 在ChromaDB中搜索 -> 返回最相关片段。

**第4页：行动扩展 - 添加Web浏览能力**
- **标题：** 让Agent“上网冲浪” - Web浏览行动
- **要点：**
    - **行动目标：** 获取实时信息、验证事实、抓取网页内容。
    - **实现方式：**
        - 使用`requests`库或`Playwright`/`Selenium`等自动化浏览器工具。
        - 封装为Agent可调用的函数（Tool）。
    - **示例工具定义：**
        - `Tool(name="web_search", func=search_web, description="搜索互联网获取最新信息")`
        - `Tool(name="browse_webpage", func=fetch_webpage, description="获取指定URL的文本内容")`
    - **错误处理：** 处理网络超时、404错误、反爬虫机制（如添加User-Agent）。

**第5页：行动扩展 - 添加文件操作能力**
- **标题：** 让Agent“读写文件” - 文件操作行动
- **要点：**
    - **行动目标：** 读取本地/云存储文件、写入分析结果、生成报告。
    - **常见操作：**
        - **读取：** `.txt`, `.csv`, `.pdf`, `.docx` 等格式。
        - **写入：** 创建新文件、追加内容、保存为特定格式。
    - **实现方式：**
        - 使用Python内置库（`open`）或第三方库（`pandas`, `PyPDF2`, `python-docx`）。
        - 封装为Tool，并明确描述输入输出。
    - **安全性考虑：**
        - 限制文件访问路径（沙盒化）。
        - 验证文件类型，防止恶意文件执行。

**第6页：健壮性设计 - 错误处理与异常恢复**
- **标题：** 让Agent“摔倒了能爬起来” - 错误处理与恢复
- **要点：**
    - **常见错误类型：**
        - **网络错误：** API超时、连接断开。
        - **工具错误：** 文件不存在、权限不足、解析失败。
        - **模型错误：** 输出格式不符合预期（JSON解析失败）。
    - **处理策略：**
        - **重试机制：** 对临时性错误（如网络波动）进行有限次重试（指数退避）。
        - **降级处理：** 如果主要工具失败，尝试备用工具或返回友好提示。
        - **日志记录：** 记录错误详情，便于调试和监控。
    - **恢复流程示例：**
        1. Agent尝试调用`browse_webpage`失败。
        2. 捕获`TimeoutError`。
        3. 记录日志：“访问example.com超时”。
        4. 向用户返回：“抱歉，我暂时无法访问该网页，请稍后再试或检查网址。”
