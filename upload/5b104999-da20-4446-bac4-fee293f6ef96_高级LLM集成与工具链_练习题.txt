## 高级LLM集成与工具链练习题

### 选择题

**1. 关于 Chain-of-Thought (CoT) Prompt Engineering 技术，以下哪个描述最准确？**
- A. CoT 技术通过随机生成多个答案来提高模型输出的多样性。
- B. CoT 技术通过在提示中提供一系列中间推理步骤，引导模型解决复杂问题。
- C. CoT 技术仅适用于数学计算问题，对其他类型的推理任务无效。
- D. CoT 技术需要模型具有外部知识库的访问权限才能生效。

**正确答案：B**

**2. 在 Function Calling 技术中，如果 LLM 识别出需要调用外部函数（例如获取天气数据），它会如何响应？**
- A. 直接执行该函数并返回结果。
- B. 返回一个包含函数名称和参数的 JSON 结构，由开发者执行并回传结果。
- C. 拒绝生成响应，要求用户手动输入数据。
- D. 忽略函数调用请求，继续生成文本回答。

**正确答案：B**

**3. 在实现工具调用（Tool Use）时，以下哪个是“将外部 API 集成到 LLM 工作流中”的核心挑战？**
- A. 确保 API 的调用速度足够快，不影响用户体验。
- B. 将外部 API 返回的结构化数据（如 JSON）转换为 LLM 可理解的上下文，并处理可能的错误或异常。
- C. 为每个 API 编写独立的 Prompt，避免通用性。
- D. 避免使用任何第三方 API，仅依赖 LLM 自身知识。

**正确答案：B**

**4. 关于 LLM 的“记忆管理（Memory）”和“上下文窗口优化”，以下哪个说法是正确的？**
- A. 上下文窗口越大越好，可以无限扩展记忆容量。
- B. 记忆管理通常通过滑动窗口、摘要或向量数据库等技术，在有限的上下文窗口内保留关键信息。
- C. 优化上下文窗口意味着固定上下文长度，不允许动态调整。
- D. 记忆管理只对长对话有效，对单次问答没有影响。

**正确答案：B**

### 简答题

**5. 请简述如何通过 Prompt Engineering 的高级技巧（如 Chain-of-Thought）结合 Function Calling，实现一个能够回答“今天北京到上海的航班有哪些？”，并在回答后自动查询天气的智能助手流程。请描述关键步骤。**

**参考答案要点：**
1.  **初始 Prompt 设计**：编写一个包含系统提示（System Prompt）的 Prompt，明确告诉 LLM 它可以调用两个函数：`search_flights(origin, destination, date)` 和 `get_weather(city, date)`。同时，使用 Chain-of-Thought 技巧，要求 LLM 在回答前先“思考”并分析用户意图。
2.  **用户提问**：用户输入“今天北京到上海的航班有哪些？并在回答后帮我查一下北京的天气。”
3.  **LLM 推理与函数调用**：LLM 识别出需要先调用 `search_flights("北京", "上海", "今天")` 来获取航班信息。它不会直接执行，而是返回一个包含函数名称和参数的 JSON 结构给开发者。
4.  **开发者执行函数**：开发者（或框架）接收到这个 JSON，调用真实的航班查询 API，获取航班列表（如 JSON 数组）。
5.  **结果回传**：开发者将 API 返回的航班数据（如“航班有：CA1234, MU5678”）作为新的上下文（Assistant 消息或 Tool 消息）回传给 LLM。
6.  **LLM 生成回答**：LLM 基于这个结果生成自然语言回答：“今天北京到上海的航班有：CA1234（8:00-10:30）和 MU5678（14:00-16:20）。” 同时，LLM 再次触发 CoT 推理，识别出“查天气”的任务，并调用 `get_weather("北京", "今天")`。
7.  **第二次函数调用**：开发者执行天气 API 查询，获取天气数据（如“晴，15-25°C”）。
8.  **最终回答**：开发者将天气数据回传，LLM 生成最终回答：“另外，今天北京的天气是晴，温度15-25°C。” 整个过程通过 Chain-of-Thought 确保了逻辑的连贯性，而 Function Calling 则实现了与外部系统的交互。
