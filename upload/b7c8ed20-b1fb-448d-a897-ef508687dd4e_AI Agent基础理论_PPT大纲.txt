## 第2页：什么是 AI Agent？
- AI Agent 是能够感知环境、自主决策并采取行动的智能实体
- 核心特征：
  - **自主性**：无需人类持续干预
  - **反应性**：能感知环境变化并响应
  - **目标导向**：拥有明确目标或任务
  - **社交能力**：可与其他 Agent 或人类交互
- 应用场景：自动驾驶、智能客服、游戏NPC、自动化工作流

---

## 第3页：Agent 类型（一）：Reactive Agent
- **Reactive Agent（反应式Agent）**
  - 基于当前感知直接做出反应，不依赖内部状态或历史记忆
  - 典型结构：感知 → 条件规则 → 行动
  - 优点：简单、响应快、计算开销低
  - 缺点：缺乏长期规划与记忆
- 示例：真空吸尘器 Agent、简单游戏中的敌人AI

---

## 第4页：Agent 类型（二）：Deliberative Agent
- **Deliberative Agent（审慎式Agent）**
  - 拥有内部世界模型，能够进行推理与规划
  - 典型结构：感知 → 状态更新 → 推理/规划 → 行动
  - 优点：可处理复杂任务、具备长远目标
  - 缺点：计算复杂度高、实时性差
- 示例：机器人路径规划、智能调度系统

---

## 第5页：认知架构：感知-行动循环
- **感知-行动循环**是 Agent 认知架构的基础
  - 感知环境 → 状态更新 → 决策/推理 → 执行行动 → 反馈循环
- 关键组件：
  - **传感器/感知模块**：获取环境数据
  - **状态表示**：内部模型或记忆
  - **决策引擎**：规则、搜索、强化学习等
  - **执行器/行动模块**：改变环境
- 示例图：感知 → 推理 → 行动 → 环境反馈

---

## 第6页：多Agent系统基础
- **多Agent系统（MAS）**：多个Agent协同工作完成任务
- 核心概念：
  - **通信**：Agent之间交换信息
  - **协调**：避免冲突、资源分配
  - **协作**：共同完成复杂目标
  - **竞争**：如拍卖、博弈场景
- 常见架构：
  - 集中式/分布式/混合式
  - 黑板系统、合同网协议
- 应用：无人机编队、分布式传感器网络、交通流量管理

---

## 第7页：主流Agent框架原理
- **LangChain**
  - 基于语言模型的Agent框架，支持工具调用、记忆、链式推理
  - 核心组件：LLM、工具、Agent执行器、记忆模块
  - 典型应用：聊天机器人、代码生成、数据查询
- **AutoGPT**
  - 自主目标驱动的Agent，可分解任务、调用工具、自我迭代
  - 工作流：设定目标 → 分解子任务 → 执行 → 反馈 → 调整
  - 特点：长期记忆、互联网访问、文件操作
- 对比：LangChain更灵活可定制，AutoGPT更强调自主性

## 第8页：总结与展望
- AI Agent 是构建智能系统的核心单元
- 从简单反应式到复杂审慎式，再到多Agent协作，能力逐步增强
- 主流框架（LangChain、AutoGPT）加速了Agent应用落地
- 未来趋势：
  - 更强的推理与规划能力
  - 多模态感知与行动
  - 人机协作与安全对齐
