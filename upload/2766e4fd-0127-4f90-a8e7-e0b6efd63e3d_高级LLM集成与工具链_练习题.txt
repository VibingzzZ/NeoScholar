### 高级LLM集成与工具链练习题

#### 一、选择题（每题4个选项，标注正确答案）

**1. 在Prompt Engineering中，Chain-of-Thought（CoT）技术的主要目的是什么？**  
A. 减少模型的输出长度以提高速度  
B. 引导模型逐步推理，提升复杂问题的准确性  
C. 强制模型仅输出JSON格式的数据  
D. 将多轮对话压缩为单次请求  
**正确答案：B**

**2. 在Function Calling技术中，当模型返回一个函数调用请求时，开发者应如何处理？**  
A. 直接执行该函数并返回结果给模型  
B. 忽略该请求，仅生成文本回复  
C. 将函数调用请求嵌入到系统提示中  
D. 等待模型自动调用外部API  
**正确答案：A**

**3. 关于工具调用（Tool Use）与外部API集成，以下哪项描述是正确的？**  
A. 工具调用只能在模型训练阶段使用  
B. 模型可以直接访问外部数据库，无需开发者干预  
C. 开发者需要定义工具的描述和参数格式，供模型选择调用  
D. 工具调用会显著降低模型的推理速度，因此应避免使用  
**正确答案：C**

**4. 在记忆管理（Memory）中，以下哪种方法最适合处理长对话场景下的上下文窗口限制？**  
A. 每次请求都发送完整的对话历史  
B. 使用滑动窗口机制，仅保留最近的N轮对话  
C. 将所有对话内容编码为固定长度的向量  
D. 忽略历史信息，仅依赖当前输入  
**正确答案：B**

**5. 上下文窗口优化中，以下哪项策略可以有效减少Token消耗？**  
A. 使用更长的系统提示以增强模型理解  
B. 将历史对话摘要化，替换原始详细内容  
C. 每次请求都包含完整的历史记录  
D. 增加max_tokens参数以生成更长回复  
**正确答案：B**

#### 二、简答题

**1. 简述Chain-of-Thought（CoT）提示技术的实现步骤，并举例说明其在数学推理问题中的应用。**  
**参考答案：**  
- 步骤：在提示中明确要求模型分步思考，例如“让我们一步一步推理”。  
- 示例：问题“小明有5个苹果，给了小红2个，又买了3个，现在有几个？”  
  - 模型输出：  
    - 第一步：初始有5个苹果。  
    - 第二步：给小红2个，剩余5 - 2 = 3个。  
    - 第三步：又买了3个，现在有3 + 3 = 6个。  
    - 答案：6个。

**2. 在Function Calling中，如何设计一个函数定义，使模型能够调用天气API获取指定城市的温度？请写出一个JSON Schema示例。**  
**参考答案：**  
```json
{
  "name": "get_weather",
  "description": "获取指定城市的当前温度",
  "parameters": {
    "type": "object",
    "properties": {
      "city": {
        "type": "string",
        "description": "城市名称，例如北京、上海"
      }
    },
    "required": ["city"]
  }
}
```
- 开发者需将此定义传递给模型，模型在需要时返回函数调用请求，开发者执行API并返回结果。

**3. 解释记忆管理中的“滑动窗口”机制如何工作，并说明其优缺点。**  
**参考答案：**  
- 工作原理：只保留对话中最近的N轮交互（如最近10轮），丢弃较早的历史。  
- 优点：有效控制Token消耗，避免超出上下文窗口限制，适合长对话场景。  
- 缺点：可能丢失早期关键信息（如用户偏好或约定），导致模型无法完全理解上下文。

**4. 在设计一个支持工具调用的LLM应用时，开发者需要定义哪些关键组件？请列出至少三个。**  
**参考答案：**  
- 工具描述：每个工具的名称、用途和参数格式（如JSON Schema）。  
- 工具执行逻辑：开发者编写的后端函数，用于处理模型返回的调用请求。  
- 结果反馈机制：将工具执行结果（如API返回值）以特定格式回传给模型，供其生成最终回复。
