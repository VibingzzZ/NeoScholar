#### **第2页：从后端视角看前端——页面是什么？**

*   **标题：** 重新认识“页面”
*   **要点：**
    *   **传统后端渲染 (如Django/Jinja2)：** 服务器拼接HTML字符串，返回完整页面。HTML结构由后端模板控制。
    *   **前后端分离 (如RESTful API + Vue/React)：**
        *   **后端：** 只负责提供JSON数据（您熟悉的领域）。
        *   **前端：** 负责构建HTML结构、CSS样式，并通过JS获取数据渲染页面。
    *   **核心转变：** 前端不再是“模板”，而是一个独立的“应用程序”。HTML和CSS是它的基础。
    *   **类比：** 后端是“仓库”和“配送员”，前端是“店面”和“陈列师”。

#### **第3页：HTML5——页面的骨架 (Structure)**

*   **标题：** HTML5语义化：让代码“表里如一”
*   **要点：**
    *   **告别 `<div>` 地狱：** 使用语义化标签增强可读性和SEO。
        *   `<header>`, `<nav>`, `<main>`, `<article>`, `<section>`, `<aside>`, `<footer>`
    *   **表单新元素：** 提升用户体验，减少JS验证。
        *   `type="email"`, `type="url"`, `type="date"`, `placeholder`, `required`, `pattern`
    *   **多媒体标签：** 原生支持，无需Flash。
        *   `<video>`, `<audio>` (配合 `<source>` 标签)
    *   **实践建议：** 将页面想象成一份“报纸”，用合适的标签定义每个区域的角色。

*   **要点：**

#### **第5页：现代布局利器——Flexbox与Grid**

*   **标题：** Flexbox & Grid：一维与二维布局的王者
*   **要点：**
    *   **Flexbox (一维布局)：** 适用于导航栏、列表、居中对齐。
        *   **核心属性：** `justify-content` (主轴), `align-items` (交叉轴), `flex-wrap` (换行)
        *   **类比：** 想象一个弹性的“线”，元素可以沿这根线排列、拉伸、压缩。
    *   **Grid (二维布局)：** 适用于页面整体框架、卡片网格。
        *   **核心属性：** `grid-template-columns`, `grid-template-rows`, `grid-gap`, `grid-area`
        *   **类比：** 想象一张“表格”，你可以精确地将元素放置到指定的“单元格”中。
    *   **选择原则：** 内容决定布局用 Flexbox，布局决定内容用 Grid。

---

#### **第6页：响应式设计——适配所有屏幕**

*   **标题：** 响应式设计基础：一套代码，处处运行
*   **要点：**
    *   **核心思想：** 页面布局和内容根据设备屏幕尺寸动态调整。
    *   **三大基石：**
        1.  **流式布局 (Fluid Layouts)：** 使用百分比 (`%`) 或 `vw/vh` 单位，而非固定像素。
        2.  **弹性图片 (Flexible Images)：** `max-width: 100%; height: auto;`
        3.  **媒体查询 (Media Queries)：** 针对不同屏幕宽度应用不同CSS规则。
            ```css
            @media (max-width: 768px) {
              .container { flex-direction: column; }
            }
            ```
    *   **移动优先策略：** 先写小屏样式，再通过 `min-width` 媒体查询逐步增强。
