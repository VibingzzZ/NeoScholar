## P0 — 关键 Bug（必须最先修复）

### 1. `PathPlanningServiceImpl.generateAndSavePath()` 永远返回 `null`

- **文件**: `Backend/src/main/java/com/javaee/backend/service/impl/PathPlanningServiceImpl.java:86`
- **问题**: 方法在第 80 行成功 `insert` 学习路径后，第 86 行始终 `return null`，未返回 `path.getId()`。导致 Controller 永远收到 `null`，永远向前端返回"路径生成失败"。
- **修复**: 在第 81 行后添加 `return path.getId();`，同时将第 86 行的 `return null` 改为异常情况下的兜底返回。

```java
// 修复方案：在 insert 成功后返回 path.getId()
pathPlanningMapper.insert(path);
log.info("成功为用户{}生成包含{}个节点的专属学习路径", profile.getUserId(), pathNodes.size());
return path.getId();  // ← 添加这行
```

- **预估工时**: 0.5h

### 2. `StudentProfile` 实体使用了错误的字段类型

- **文件**: `Backend/src/main/java/com/javaee/backend/entity/StudentProfile.java`
- **问题**: `learningGoal` 和 `knowledgeBase` 字段类型为 `org.w3c.dom.Text`，`commonMistakes` 字段类型为 `dev.langchain4j.internal.Json`。这些是非常规类型，导致：
  - MyBatis-Plus 无法正确映射数据库的 TEXT 字段
  - `ProfileMergeServiceImpl` 中需要大量手动转换代码（`createText`/`createJson`/`getTextValue`/`getJsonValue`）
  - 其他服务中也需要手动调用 `.getData()` 来取值
- **修复**: 将这三个字段改为 `String` 类型，与数据库 `TEXT` 类型匹配，删除 `ProfileMergeServiceImpl` 中的手动转换方法。
- **预估工时**: 1h
