
## CI/CD

| 工作流 | 触发条件 | 内容 |
|--------|----------|------|
| `develop.yml` | push 到 `develop`/`feature/CI_CD` | 编译检查 + 非 AI 单元测试 |
| `main.yml` | push 到 `main` | 完整测试 + 打包 |

### CI 测试排除 AI 测试的方式

AI 测试（需调用大模型 API）的类名以 `AIIT` 结尾，而不是 `*Test`：

- `PathPlanningAIIT.java`
- `ProfileMergeServiceAIIT.java`
- `ResourceOrchestratorAIIT.java`

Surefire 3.5.5 的默认测试扫描模式为 `**/Test*.java`、`**/*Test.java`、`**/*Tests.java`、`**/*TestCase.java`，`AIIT` 不匹配任何模式，因此 CI 中不会被自动运行。

本地运行 AI 测试：
```bash
mvn test -Dtest='*AIIT'
```

### feature/CI_CD 分支 CI 调试记录

> 以下记录了 `feature/CI_CD` 分支 CI 测试从全量失败到全部通过的过程。

#### 问题 1: 编译错误 — 缺少类闭合括号

**现象**: `reached end of file while parsing`，测试一个都跑不了。

**根因**: `ProfileMergeServiceImpl.java` 在删除 `createJson` 方法时遗漏了类末尾的 `}`。

**修复**: 补上缺失的 `}`。

#### 问题 2: Surefire 3.5.5 排除机制全部失效

**现象**: 用了 `-Dsurefire.excludes`、`-Dtest=!*AITest`、`<excludes>`、`@Tag("ai")` + `<excludedGroups>`、`assumeTrue` 五种方式试图跳过 AI 测试，**全部不生效**，AI 测试依然被执行，因无有效 API Key 返回 401 导致 BUILD FAILURE。

**根因**: Maven Surefire Plugin 3.5.5 的排除机制与 Spring Boot 3.5 的 parent POM 存在兼容性问题，所有配置层面的排除均被覆盖或忽略。

**最终方案**: 将 AI 测试类名从 `*AITest` 改为 `*AIIT`，利用 Surefire 的默认文件名匹配规则绕过扫描——不匹配任何模式就不会被运行。

#### 其他已修复问题（feature/CI_CD 分支上）

| 问题 | 修复 |
|------|------|
| 重复 `GlobalExceptionHandler` → Bean 名称冲突 | 删除 `exception/GlobalExceptionHandler.java` |
| MyBatis-Plus 配置指向不存在的 `com.example` 路径 | 移除 `mapper-locations`，修正 `type-aliases-package` |
| `GenerateTools` ↔ `ResourceOrchestratorService` 循环依赖 | 加 `@Lazy` 注解 |
| `StudentProfile.commonMistakes` 使用 `dev.langchain4j.internal.Json` | 改为 `String` 类型 |
| `application.yaml` 中 `api-key: ${QWEN_API_KEY}` 无默认值 | 改为 `${QWEN_API_KEY:}` 允许空值启动 |
| Windows/Linux 换行符不一致 | 添加 `.gitattributes` 强制 LF |
